# Security Policy

## Supported Version

Security fixes target the `main` branch.

## Reporting

Do not open public issues with exploit details, private files, credentials, or personal information.

Use GitHub private vulnerability reporting if it is enabled for the repository. If private reporting is unavailable, open a minimal public issue asking for a maintainer contact and leave out sensitive details.

## Scope

Useful security reports include:

- Dependency vulnerabilities that affect the shipped app or Electron wrapper.
- Unsafe Electron protocol, file, or media handling.
- Secret exposure in tracked files.
- Cross-site scripting or unsafe HTML injection paths.

Gameplay bugs, rendering glitches, and balance issues should use normal GitHub issues.
